Fix: gate the joint votes (at minimum the scale2 vote) on the same Schur-complement conditioning solve_lever_arm uses, or document the >=3-distinct-turn-type excitation requirement in calibration.hpp/DESIGN (it currently lives only in a test comment) and verify on real data.
- src/core/calibration.cpp:967: MAJOR: the range guard makes a legitimate TRUE residual scale outside (range_min, range_max) — default (0.5, 1.5), so any >50% scale error; 1.6 is OUT, 1.4 is fine — NEVER vote, never commit, never fold. Fix: document in config.hpp (scale_hist + joint_lever_scale) that scale_hist's range bounds the turn-path's recoverable residual per fold, and surface a skipped-vote diagnostic so out-of-regime is distinguishable from unexcited.
