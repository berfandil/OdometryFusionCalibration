src/core/calibration.cpp:287,328-338: MAJOR - extrinsic-rotation bootstrap is NON-CONTRACTIVE from a large basepoint error (the headline limitation, as flagged — CONFIRMED). observe() votes `dR = rotation_between(e_x, g_obs)` so `dR·e_x = g_obs` where `g_obs = R_basepoint·dir_B`; extrinsic() then returns `X.R = dR_mode·R_basepoint`. The recovered map satisfies `X.R·dir_B = dR·g_obs`, which equals `e_x` ONLY when `g_obs ≈ e_x` (basepoint ≈ truth) — away from the fixed point the composition is applied in the WRONG direction, so the readout is biased by ≈ the basepoint error itself. Feeding extrinsic() back as the new basepoint (re-anchor) therefore amplifies, not cancels, the error → walks away. CONTAINED FIX EXISTS: vote/read the INVERSE minimal rotation — `rotation_between(g_obs, e_x)` (or equivalently `X.R = dR_mode.transpose()·R_basepoint`), giving `X.R·dir_B = rot(g_obs→e_x)·g_obs = e_x` exactly, making the re-anchor contractive.
