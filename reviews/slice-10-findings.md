src/core/estimator.cpp:989-1006: MAJOR - under per-source DROPOUT the smoothed twist is paired with a NON-time-aligned consensus (silent calibration bias — the exact pillar-2 failure mode, as flagged "approximate"). The smoother's emitted sample is the (slot.count − L)-th PUSH for that slot, but slot.count only advances on steps where the source is BOTH covered AND smoothing-enabled (estimator.cpp:958-963), while `calib_ring[0]` advances every deeper-branch step (every fuse). If a smoothing source misses any step in the window (t−L, t] but WAS present at t−L (so it is in `old.ids` and `smoother.ready(slot)` holds), the emitted smoothed twist corresponds to a wall-clock step EARLIER than t−L yet is fed alongside `old.fused_omega`/`old.fused_motion`/`old.dt` from step t−L. On a turning trajectory the twist differs between steps, so this injects a phase error ≈ (missed-step-count · slope · dt) into the very vote zero-phase was meant to de-bias — it does NOT NaN/crash, it silently biases (and `old.dt` is then also the wrong step's dt for the `exp(w·dt_old)` reconstruction). The brief's "approximate" is therefore a real silent-bias path, not merely degraded variance. Concrete fix: stamp each smoother sample with the calib_ring frame index it was pushed for (or track per-slot push-count-at-frame-creation in the CalibFrame), and at consumption use the smoothed value ONLY when (current push count − stored push count) == L; otherwise fall back to `old.reported[i]` (raw delayed delta — already the dropout fallback). That makes the substitution byte-correct under arbitrary per-source coverage and downgrades dropout to a clean variance loss instead of a bias.
